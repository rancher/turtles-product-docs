== 認定プロバイダとは何ですか？

認定プロバイダは、TurtlesチームのCIワークフローによって**積極的にテストされている**CAPIプロバイダであり、RancherのCAPI拡張機能と適切に動作することを検証しています。ただし、ほとんどのCAPIプロバイダはKubernetesオープンコミュニティによって維持されているアップストリームプロジェクトであるため、**新しいプロバイダのバージョンがリリースされる際に**問題が発生しないことを保証することはできません。

== 認定プロバイダのためのクラスタークラスサポート

以下は、各認定プロバイダとそのクラスター トポロジー機能のサポートに関するサポートマトリックスです。

[tabs]
======
CAPZ::
+
--
- **完全なサポート** `ClusterClass`：管理された（AKS）および管理されていない（仮想マシン）クラスターの両方がトポロジーを介してプロビジョニングできます。
--

CAPA::
+
--
- **完全なサポート** `ClusterClass`：管理された（EKS）および管理されていない（EC2ベース）クラスターの両方がトポロジーを介してプロビジョニングできます。
--

CAPG::
+
--
- **完全なサポート** `ClusterClass`：管理された（GKE）および管理されていない（仮想マシン）クラスターの両方がトポロジーを介してプロビジョニングできます。
--

CAPRKE2::
+
--
- **完全なサポート** `ClusterClass`.
--

CABPK::
+
--
- **完全なサポート** `ClusterClass`.
--

CAPV::
+
--
- **完全なサポート** `ClusterClass`.
--

CAPD::
+
--
- **完全なサポート** `ClusterClass`.
--
